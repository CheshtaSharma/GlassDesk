Nothing yet. Upload a file, select it, and ask a question — every step the AI takes will stream in here as it happens.

Any file is parsed once on upload. For PDFs longer than a few pages, the model first skims every page to detect chapter boundaries on its own, then text is split into chunks per chapter, embedded, and stored in SQL Server (large files process in the background; the progress line above shows real status, not a spinner). The full document is never sent to the model when answering — it can only request a chapter overview, a chapter's chunk index, a specific chunk by id, a keyword search, or a FAISS semantic search, and each request/response above is the complete context exchange. The loop stops the instant the AI answers.
